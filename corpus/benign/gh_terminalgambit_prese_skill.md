# Skill: Presentation Framework

## Description
Build branded HTML slide decks from a `presentation.yaml` config + `metrics.json` data file. The framework uses Jinja2 templates with a shared CSS theme to generate individual HTML slides and an iframe-based navigator with keyboard controls.

## Prerequisites
- Python 3.10+
- The `presentation-framework` package installed: `pip install -e /path/to/presentation-framework`
- A `metrics.json` data file (required — never hardcode data)

## Quick Start
```bash
# Scaffold a new project
pf init my-deck

# Edit my-deck/presentation.yaml and my-deck/metrics.json

# Build slides (add --open to auto-launch in browser)
pf build --config my-deck/presentation.yaml --metrics my-deck/metrics.json --output my-deck/slides --open

# Preview
pf serve --dir my-deck/slides

# Package for sharing
pf zip --dir my-deck/slides
```

## Core Rule
**All data must flow from `metrics.json`**. Use `{{ metrics.path.to.value }}` interpolation in YAML strings — never hardcode numbers, counts, or dynamic values.

## File Structure
A presentation project contains:
```
my-deck/
├── presentation.yaml   # Slide definitions (layouts, content, metrics refs)
├── metrics.json        # All data: numbers, benchmarks, costs, etc.
└── slides/             # Built output (generated by `pf build`)
    ├── slide_01.html
    ├── slide_02.html
    ├── present.html    # Navigator — open this in browser
    └── theme/          # CSS files (copied from framework)
```

## presentation.yaml Structure
```yaml
meta:
  title: "Deck Title"
  authors: ["Author 1", "Author 2"]
  institution: "School or Company"
  date: "2026-01-01"

theme:
  primary: "#1C2537"      # Background color — any hex value
  accent: "#C4A962"       # Accent color — any hex value
  fonts:
    heading: "Playfair Display"
    subheading: "Montserrat"
    body: "Lato"

slides:
  - layout: title         # Layout template name
    data:                  # Data passed to the template
      title: "My Title"
      subtitle: "Subtitle"
```

## Theme Customisation
The `theme` section in `presentation.yaml` **fully controls the color scheme**. The builder generates a custom `variables.css` at build time — all accent variants (dim, glow, border, background) are automatically derived from the hex colors you provide.

To match a company's brand, just change `primary` and `accent`:
```yaml
# Example: dark green + white-gold for a luxury brand
theme:
  primary: "#0D1F0D"
  accent: "#D4AF37"
  fonts:
    heading: "Cormorant Garamond"
    subheading: "Raleway"
    body: "Source Sans Pro"
```
The builder derives all CSS custom properties (`--pf-accent-dim`, `--pf-accent-glow`, `--pf-card-border`, etc.) from the hex accent color, so every component automatically adapts.

Fonts are loaded from Google Fonts CDN — use any family name available there.

## Metrics Interpolation
Any string value in `data:` can reference metrics using `{{ metrics.path.to.value }}`:
```yaml
# metrics.json: {"summary": {"total_assets": 128, "categories": 9}}
slides:
  - layout: title
    data:
      title: "{{ metrics.summary.total_assets }} Assets"
      # Renders as: "128 Assets"
```
Interpolation works inside any string at any nesting depth — titles, bullets, table cells, etc.

## Available Layouts

### 1. `title` — Opening / hero slide
Centered layout with decorative frame, large title, and optional feature grid.
```yaml
- layout: title
  data:
    title: "JEMA"                     # Required — large heading
    subtitle: "Subtitle Here"         # Optional — uppercase subheading
    tagline: "One-line description"   # Optional — body text
    quote: "\"Quoted text.\""         # Optional — italic accent text
    title_size: "120px"               # Optional — default 120px
    features:                         # Optional — bottom feature grid
      - icon: gem                     # FontAwesome icon name (no "fa-" prefix)
        label: "Feature Label"
        sub: "Description"
    footer: "Author — Institution"    # Optional — bottom text
```

### 2. `two-column` — Main content layout
Two-column grid with header. Each column contains a list of typed content blocks.
```yaml
- layout: two-column
  data:
    title: "Slide Title"             # Required
    subtitle: "Optional subtitle"
    column_ratio: "2fr 1fr"          # Optional — CSS grid-template-columns
    left:                            # List of content blocks
      - type: card
        icon: layer-group
        title: "Card Title"
        text: "Card body text."
        bullets:
          - "Bullet point with <strong>HTML</strong> support"
      - type: solution-box
        badge: "Badge Text"
        title: "Box Title"
        icon: gem
        items:
          - "Checklist item 1"
          - "Checklist item 2"
      - type: stat-grid
        cols: 2                      # 2, 3, or 4
        stats:
          - { value: "128", label: "Assets" }
          - { value: "9", label: "Categories" }
      - type: table
        section_title: "Table Section"
        section_icon: "chart-bar"
        headers: ["Col A", "Col B"]
        rows:
          - ["Row 1A", "Row 1B"]
        winner_rows: [0]             # Highlighted row indices
        footnote: "Note text"
      - type: dist-bars
        section_title: "Distribution"
        bars:
          - { label: "Category A", width: 80, pct: "80%", color_class: "bar-gold" }
      - type: insight
        text: "<strong>Key insight</strong> — explanation here."
        icon: lightbulb
      - type: value-prop
        title: "Value Title"
        text: "Value description."
      - type: takeaway
        text: "Key takeaway text."
      - type: html
        content: "<div>Raw HTML</div>"
    right:                           # Same block types as left, plus:
      - type: unit-grid
        tiers:
          - { tier: "Basic", price: "$10", unit: "per month" }
      - type: val-bars
        items:
          - { label: "Metric", lift: "+12%", width: 75, color: "#C4A962" }
```

### 3. `three-column` — Multi-column content
Three-column grid. Uses `columns` (list of 3 lists) instead of `left`/`right`.
```yaml
- layout: three-column
  data:
    title: "Slide Title"
    column_ratio: "1fr 1fr 1fr"      # Optional
    columns:
      - # Column 1 — list of blocks
        - type: card
          header: "Card Header"
          icon: chart-line
          content: "<p>HTML content</p>"
      - # Column 2
        - type: stat-grid
          stats:
            - { value: "42", label: "Things" }
      - # Column 3
        - type: image
          src: "path/to/image.png"
          alt: "Description"
```
Supported block types: `card`, `stat-grid`, `table`, `val-bars`, `insight`, `image`, `html`.

### 4. `data-table` — Benchmarks and comparisons
Two-section layout optimized for tables, bars, and analysis. Uses `sections` list.
```yaml
- layout: data-table
  data:
    title: "Benchmark Results"
    subtitle: "Model comparison"
    column_ratio: "1fr 1fr"          # Optional
    sections:
      - section_title: "Section A"
        section_icon: "vector-square"
        table:
          headers: ["Model", "Score", "Notes"]
          rows:
            - ["Model A", "0.95", "Best"]
            - ["Model B", "0.87", "Good"]
          winner_rows: [0]
          total_row: ["Total", "—", "—"]  # Optional
          footnote: "Footnote text"
        bars:                        # Optional — dist_bars
          - { label: "A", width: 95, pct: "95%" }
        unit_tiers:                  # Optional — unit_grid
          - { tier: "Tier 1", price: "$50", unit: "per run" }
        insight:
          text: "Key finding."
          icon: lightbulb
      - section_title: "Section B"
        table:
          headers: ["Prompt", "Accuracy"]
          rows:
            - ["v1", "54.2%"]
```

### 5. `stat-grid` — KPI / metrics dashboard
Two-column layout with stat grids, cards, and insights.
```yaml
- layout: stat-grid
  data:
    title: "Key Metrics"
    column_ratio: "1fr 1fr"
    columns:
      - # Column 1
        - type: stat-grid
          cols: 2
          stats:
            - { value: "128", label: "Total Assets" }
            - { value: "9", label: "Categories" }
        - type: insight
          text: "Important note."
      - # Column 2
        - type: card
          header: "Details"
          icon: info-circle
          items:                     # Rendered as checklist
            - "Item one"
            - "Item two"
        - type: section-title
          text: "Section"
          icon: chart-bar
```

### 6. `closing` — Thank you / Q&A slide
Centered layout with decorative frame (same as title).
```yaml
- layout: closing
  data:
    title: "Thank You"
    subtitle: "Questions & Discussion"
    title_size: "72px"               # Optional — default 72px
    info_items:                      # Optional — row of pills/info
      - type: pill
        icon: "fa-brands fa-github"  # Full FA class for pills
        text: "github.com/user/repo"
      - type: info
        icon: "fa-solid fa-graduation-cap"
        text: "Institution Name"
    footer: "Footer text"
```

## FontAwesome Icons
Icons use FontAwesome 6 Free. In `data:` fields, use just the icon name without prefixes:
```yaml
icon: gem          # Renders as: <i class="fas fa-gem"></i>
icon: chart-line   # Renders as: <i class="fas fa-chart-line"></i>
```
Exception: `closing` layout `info_items` use full FA class strings (e.g., `"fa-brands fa-github"`).

## Custom HTML Blocks
Any layout that accepts content blocks supports `type: html` for raw HTML:
```yaml
- type: html
  content: |
    <div style="padding: 10px; border: 1px solid var(--pf-accent);">
      Custom content here
    </div>
```

## Step-by-Step: Creating a New Presentation
1. **Prepare `metrics.json`** — Populate with all data the slides will reference
2. **Write `presentation.yaml`** — Define slides using layouts above, referencing `{{ metrics.x.y }}` for all dynamic values. Set `theme.primary` and `theme.accent` to match the brand.
3. **Build** — `pf build --config presentation.yaml --metrics metrics.json --output slides --open`
4. **Verify** — Browser opens automatically. Use arrow keys to navigate, `G` for grid overview, `F` for fullscreen
5. **Iterate** — Edit YAML, rebuild, refresh browser
6. **Share** — `pf zip --dir slides` creates a `.zip` file that anyone can extract and open

## CLI Reference
- `pf init <name>` — Scaffold a new project
- `pf build -c <yaml> -m <json> -o <dir> [--open]` — Build slides; `--open` auto-launches browser
- `pf serve -d <dir> -p <port>` — Local preview server
- `pf zip -d <dir> [-o <path>]` — Package slides into a shareable `.zip`
