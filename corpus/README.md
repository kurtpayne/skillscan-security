# SkillScan Security — Training Corpus

This directory contains labelled skill files used to fine-tune and evaluate the
`protectai/deberta-v3-base-prompt-injection-v2` ML detector.

## Directory layout

```
corpus/
  benign/            ← Clean skill files (label: 0 / BENIGN)
  malicious/         ← Skill files with malware/exfil payloads (label: 1 / INJECTION)
  prompt_injection/  ← Skill files with prompt-injection attacks (label: 1 / INJECTION)
  manifest.json      ← Auto-generated by CorpusManager; do not edit by hand
```

## Labelling conventions

Each file must be a valid Markdown file (`.md`) or plain text (`.txt`).
The label is inferred from the subdirectory:

| Directory | Label | Numeric |
|---|---|---|
| `benign/` | BENIGN | 0 |
| `malicious/` | INJECTION | 1 |
| `prompt_injection/` | INJECTION | 1 |

## Adding new examples

1. Drop the file into the appropriate subdirectory.
2. Run `skillscan corpus sync` to update `manifest.json` and check the delta threshold.
3. If the delta threshold is crossed, the CI workflow will schedule a fine-tune run.

Alternatively, push directly — the `corpus-sync.yml` GitHub Actions workflow runs
`skillscan corpus sync --check` on every push that touches `corpus/**` and opens
a fine-tune dispatch if the threshold is met.
