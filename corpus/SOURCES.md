## Source Details

### 1. GitHub Public Repositories (`benign/`)

**Description:** Legitimate agent skill instruction files scraped from public GitHub
repositories via the GitHub Search API. Includes SKILL.md, CLAUDE.md,
INSTRUCTIONS.md, system_prompt.md, skill.yaml, and .mcp.json file patterns.

**Collection scripts:** `scripts/scrape_github_skills.py`,
`scripts/expand_corpus_round3.py` — 33 query variants, up to 1,000 results per query.

**Filename convention:** Files scraped from GitHub use the prefix `gh_` followed by
a slug of the source repo and filename (e.g., `gh_0xartex_agentwallet__skill.md`).

**License:** Files are from public repositories and retain their original licenses.
Used here under fair use for security research purposes. No credentials, secrets, or
private data are included — files are public skill/instruction definitions only.

**Attribution:** Original authors and repository owners of each scraped file.

---

### 2. Hand-crafted Injection Examples (`prompt_injection/`, `malicious/`, `social_engineering/`, `graph_injection/`)

**Author:** kurtpayne ([@kurtpayne](https://github.com/kurtpayne))

**License:** MIT (same as this repository)

**Description:** Manually authored examples covering the full attack taxonomy:

- **`prompt_injection/`** (`pi01_*` through `pi22_*`, `a23_*`): Direct instruction
  override, role hijacking, context extraction, fake tool output, indirect injection
  via external data, persona hijacking, goal hijacking, exfil via output fields,
  version field injection, social engineering prerequisites, credential exfiltration,
  reverse shell embedding, multi-turn memory injection, typosquatting changelogs,
  nested skill abuse, permission escalation, error message leakage, supply chain
  dependency injection, webhook exfiltration disguised as logging.

- **`malicious/`** (`a01_*` through `a32_*`): Supply-chain malware patterns —
  curl|bash remote execution, DNS exfiltration, SMTP exfiltration, cloud upload
  exfiltration, privilege escalation, trojan source, and named-campaign-inspired
  examples. The named-campaign files (ClawHavoc, BlokTrooper, GlassWorm, CursorJack,
  ClickFix, Deno BYOR, OpenClaw gatewayUrl) are **original hand-crafted examples
  inspired by public incident reports** — they are not copies of real malicious
  files. The underlying campaign names and attack patterns are drawn from publicly
  disclosed security research (Koi/ClawHavoc, Snyk/clawdhub, Ethiack CVE-2026-25253).

- **`social_engineering/`** (`se_*` and `benign_*`): Social engineering injection
  patterns (account expiry harvesting, authority impersonation, browser cookie
  harvesting, coercive refusal, compliance audit harvesting, developer onboarding
  harvesting, devops emergency, direct token paste, env dump disguised as setup,
  fake 2FA bypass) alongside hard-negative benign counterparts (API key docs,
  credential rotation guides, security setup instructions that mention sensitive
  paths without malicious intent).

- **`graph_injection/`** (`PINJ-GRAPH-001` through `PINJ-GRAPH-004`): Multi-file
  skill graph injection scenarios where no single file looks dangerous but the
  combined skill graph forms an exploit chain.

### 3. Synthetic Augmentation (`augmented/`)

**Author:** SkillScan project (generated by Manus AI assistant during corpus
expansion, reviewed by kurtpayne)

**License:** MIT (same as this repository)

**Method:** `scripts/augment_corpus.py` — benign skill files from `corpus/benign/`
are modified by appending realistic injection payloads drawn from a curated template
library (persona hijack, instruction injection, boundary attack, incremental
compliance, hypothetical framing). Augmented files are labeled injection (label 1).

**Filename convention:** `aug_{original_slug}_{attack_type}_v{n}.md`

**Note:** Augmented examples are lower quality than hand-crafted ones — they are
useful for class balancing but should not dominate the injection training set.
The `BENCHMARK_TEST_ONLY` metadata comment was stripped from all files before
training to prevent label leakage.

### 4. zast-ai/skill-security-reviewer (`benchmark_injection/`)

**Repository:** https://github.com/zast-ai/skill-security-reviewer

**Author:** chris (zast-ai) and contributors

**License:** MIT License — https://github.com/zast-ai/skill-security-reviewer/blob/main/LICENSE

**Description:** A benchmark test suite of 100 neutralized malicious skill samples
covering 8 attack categories (THEFT, EXEC, PERSIST, EXFIL, INJ, ABUSE, DECEP,
SUPPLY) and 50 advanced obfuscated samples (base64, hex, unicode, AES, XOR,
multi-layer encoding, etc.). All samples use `[REDACTED]` placeholders and the
`BENCHMARK_TEST_ONLY` marker — they cannot execute malicious behavior and are
intended for security testing only.

**Corpus directories used:**
- `malicious-skill-benchmark/` → 100 samples (benchmark-001 through benchmark-100)
- `advanced-malicious-skills/` → 50 samples (adv-001 through adv-050)

**Import notes:**
- The `<!-- BENCHMARK_TEST_ONLY ... -->` HTML comment marker was stripped from file
  content before import to prevent label leakage into the model's input features.
- Files are stored as `{original-dir-name}.md` in `corpus/benchmark_injection/`.

### 5. Liu et al. 2026 — Malicious Agent Skills in the Wild (not pursued)

**Paper:** "Malicious Agent Skills in the Wild: A Large-Scale Security Empirical Study"

**arXiv:** https://arxiv.org/abs/2602.06547

**Authors:** Y. Liu et al. (Quantstamp, Fujian Normal University)

**Description:** The paper describes a three-tiered labeled dataset of agent skill
security covering 98,380 skills, including 157 behaviorally-verified malicious skills
(Tier 3). The dataset would provide out-of-distribution injection examples relative
to SkillScan's hand-crafted corpus.

**Status:** Not pursued. The dataset requires direct author access and the timeline
is uncertain. The preferred path for improving injection recall is (a) the behavioral
sandbox (`skillscan-trace`, Milestone 18) to generate ground-truth labels from
first-party execution traces, and (b) continued hand-crafting of diverse injection
examples across underrepresented archetypes (Agent Hijacker P1/P4, graph injection,
temporal payloads). This keeps the corpus fully under our control and avoids
dependency on third-party dataset release schedules.

## Adding New Sources

When adding a new external data source to the corpus:

1. Create a new subdirectory under `corpus/` with a descriptive name.
2. Add the source to the **Summary Table** and a **Source Details** section in this
   file with: repository/paper URL, author(s), license, description, import method,
   and a BibTeX citation block.
3. Update `scripts/finetune_modal.py` to include the new directory in `INJECTION_DIRS`
   (or the benign label path if applicable).
4. Run `python3 -c "..."` (see manifest rebuild snippet in README.md) to update
   `manifest.json`.
5. Commit both the new files and the updated `SOURCES.md` in the same PR.

If authorship is unknown (e.g., scraped files with no clear license), mark the
Author field as **"Unknown (public repository)"** and the License field as
**"Unknown / fair use (security research)"**.
